# Clipin 优雅基准(Elegance Baseline)

- 日期:2026-05-19
- 性质:**不是实施计划,是度量尺**。所有 spec/commit/review/真机验收都拿这份对照。
- 适用:全仓长期有效,跨分支跨 commit;与具体 spec 冲突时,以本文「不变量」为准,spec 只能在不违反本文的前提下细化。
- 关联:消化并固化 `2026-05-18-raycast-elegance-refinement-design.md`、`2026-05-19-ios26-liquid-glass-launcher-design.md`、根 `CLAUDE.md` 决策日志、`feat/liquid-glass-migration` 全部踩坑史。

## 0. 为什么需要这份文档(根因)

`feat/liquid-glass-migration` 有 40+ commit 在「实心深色 → 原生玻璃 → 锁深色 → 撤销锁深色 → …」之间反复。
这不是迭代,是**没有锚点的反应式打补丁**:用户真机否决 → 局部 patch → 再否决 → 再 patch。
根因不是某个技术点,是**没有一份写下来的、可证伪的「优雅长什么样 / 不优雅长什么样」**。
本文就是那个锚点:**每次改动只问一句——「这让基准线升了还是降了」**。降了或测不了,就不许进主干。

目标定义(不可再被稀释):**做一款伟大且优雅的剪贴板工具 App。** 优雅 = 三根**不可互相替代**的支柱同时达标:
代码优雅、产品优雅、界面优雅。工具类 App 体验差等于没价值——三根缺一根,App 即失败。

## 1. 代码优雅 —— 判据

**定义:单源真相;不造轮子;一眼读懂;改一处不波及全局;文件聚焦;验证机制走真实路径。**

| 优雅(达标) | 不优雅(回归) | 证伪方式 |
|---|---|---|
| 能力有单一抓手(`ClipinInk`/`ClipinSelectionInk`/`shellGap`/`shellCornerRadius`) | 同一语义散落多处魔数,改一个值调不动全局 | grep 该 token 唯一定义点 |
| 优先用系统/成熟能力(原生 `.glassEffect`/`NSGlassEffectView`/Vision/SQLite-FTS5) | 自研已被系统覆盖的能力(如曾经的 8 套手调 `ClipinGlassPalette`) | review:新增抽象必须答「为何现成方案不行」 |
| 单文件聚焦、< 500 行(硬上限 800) | `AppDelegate 1573`/`SettingsView 1165`/`PreviewPane 1006` | `wc -l`;超 800 必须有拆分任务 |
| 删除即彻底,无 compat/fallback/死代码 | 半迁移态、悬空引用、`displayName` 这类无调用方残留 | grep 被删符号 = 0 |
| 验证机制走**真实**代码路径 | QA 旁路(如 `forceSegmentHover`)短路真实路径,自检永远验不出真 bug | review:每个 QA flag 必须问「它是否绕过了被验对象」 |
| `@State`/状态在语义正确的载体上 | `@State` 挂 `ButtonStyle`(非 View)等「有时能跑」的未支持写法 | 构建 + 真实交互验证 |
| 不兜底:异常/缺失直接暴露 | `try?` 吞错、静默 default、catch-all | grep `try?` / 空 catch;review |

**红线**:任何 `*.profraw`/生成物/测试脚手架不得进 git 业务提交;QA 钩子默认全 false 且必须显式声明「非静默兜底」。

## 2. 产品优雅 —— 判据

**定义:keyboard-first 心智永不断裂;零额外操作;危险操作可逆;隐私有红线;命令入口永远可发现。**
(这是当前**最成熟**的支柱,判据用于「守住不退化」,不是「补齐」。)

| 优雅(达标) | 不优雅(回归) |
|---|---|
| ↑↓/Return/Esc/Tab/⌘1-9/Space 由 `AppDelegate` 按窗口上下文统一路由 | 任一导航退化成依赖某控件焦点(搜索框失焦后 Tab 变焦点遍历) |
| Esc 分层回退:先清搜索/筛选,回到默认列表才关面板 | Esc 直接关面板,打断探索路径 |
| 连续粘贴零额外操作:失焦自动夺回、目标 app 实时解析 | 连续粘贴需要用户额外点击/确认 |
| 删除先进 7s 可撤销窗口,退出前 finalize | 删除即硬删,图片文件同步消失不可逆 |
| `ConcealedType` 永远跳过(安全红线);Transient/AutoGenerated 由开关 | 把安全红线做成可关开关,或把降噪偏好做成不可关 |
| `⌘K` 全局命令入口在空态/无选中/筛选无果时仍可用且可见 | 空态把 `⌘K` 入口藏起来,能力退化成隐形 |
| 写剪贴板前先验 payload,失败不清空用户系统剪贴板 | 先 `clearContents()` 再发现写不了 |
| 用户可见文案全部走 `Localizable.strings`(含菜单/动态 notice/快捷键名) | 静态 key 直返英文,中文环境露馅 |

## 3. 界面优雅 —— 判据(**当前最弱、是瓶颈、是反复的源头,判据最细**)

**定义:macOS 26 原生 Liquid Glass launcher,Raycast/Spotlight 级克制;白天与夜间双境都要好;
身份/命令/内容三区视觉契约清晰;没有"看着能点其实不能"的伪控件;同心圆角;玻璃只在它该在的层。**

### 3.1 分层铁律(Apple 明文,违反即根因级错误)

- **Liquid Glass 只属于导航/控件层,内容永远坐在最底、玻璃浮其上。** 内容(列表/表格/媒体正文)**永不**上玻璃。
- **唯一合法整面玻璃** = 窗面 `NSGlassEffectView`(launcher 整窗即导航层,Spotlight 同款)。
- **底栏命令簇** = 浮在窗面玻璃之上的独立 `GlassEffectContainer`(Apple 文档化的「控件玻璃浮导航玻璃」,合法)。
- 玻璃不能采样玻璃 → 相邻玻璃必须同一 `GlassEffectContainer` 由系统融合。
- 内容区**不得**用 `ClipinContentSurface` 不透明盒子把内容从玻璃上抬走(辅助窗口除外,那是标准 grouped 面)。
- 已知有意例外(用户指令优先,非违规):预览横滚 `PreviewValueBadge` 胶囊保留 `.glassEffect`——用户多次明确要原样,review 不得据此判不合规。

### 3.2 视觉契约(可证伪,逐条真机截图核对)

| # | 优雅(达标) | 不优雅(回归) |
|---|---|---|
| V1 | 整窗连续 Liquid Glass,跟随系统明暗自适应 | 实心深色板 / 锁死 dark / 透到桌面发脏 / 发白发平 |
| V2 | 圆角逐层同心:shell→底栏簇→胶囊→键帽→选中底板,内角弧 = 外角弧 − inset | 内外角弧不同心;尖角(同心半径被算成 ~0);`.circular` 非 squircle |
| V3 | 底栏静息 = 暗克制单玻璃簇 + 极细 rim | 亮白磨砂大板 / 满宽灰底座 / 厚外框 |
| V4 | 身份锚在搜索栏左上(app 图标);底栏只承命令与上下文 | 身份放底栏左下;或多处重复出现 |
| V5 | hover 底栏控件 → 正上方派生独立玻璃胶囊(无箭头、留缝、同款暗玻璃) | 横向挤开 / tooltip / 派生簇被容器裁掉 |
| V6 | 画出来的命令都能点;非命令(面包屑/状态)不做成胶囊伪按钮 | 看着像按钮其实不可点;或该可点的只是 badge |
| V7 | 选中行任意时刻**单一**可辨;hover 明确弱于选中;玻璃面上文字清晰 | 浅毛玻璃下糊成多行同时选中;hover 残留成第二选中态 |
| V8 | accent 仅 Paste 主 CTA 一处;其余中性 | 多处 accent 稀释唯一强调点;手绘 accent 实色块 |
| V9 | 动效克制(let glass rest in steady states) | 过度缩放/位移/整列变透明等非原生动效 |
| V10 | 白天截图达标 **且** 夜间截图独立达标 | 只验一种明暗;另一境发灰/对比不足/发白 |
| V11 | 5 辅助窗口(设置/引导/权限/更新)未被主面板改动牵连 | 辅助窗口被顺带改坏或风格分叉 |

### 3.3 圆角与间距单源

- **唯一硬编码圆角** = `ClipinChrome.shellCornerRadius`(= `NSGlassEffectView.cornerRadius` = panel frame KVC,三处同源同值)。
- 其余嵌套形状用同心 API / `Capsule`;链外独立量级(card 18 / palette 26)必须在 token 注释里写明用途。
- **唯一间距 token** = `ClipinChrome.shellGap`;floating footer 让位用单一 `floatingFooterBand`(列表 inset 与预览 bottom 共用,防漂移)。

## 4. 反模式登记册(踩坑史固化 —— 再次出现即判回归,不需重新论证)

1. **玻璃放进内容层**,再被迫给列表/预览套不透明盒子"保可读"——v1 根因,三症状(大框/不原生/底栏脏)同源。
2. **实心深色窗面**冒充 launcher——被用户多轮真机否决,"不够原生"。回归此路即错。
3. **锁死 dark / 不跟随系统**——白天境必然发平;明暗必须各自独立达标。
4. **mock 验材质**——`backdrop-filter` ≠ 真 `.glassEffect`,mock 只能定布局,材质验收唯一口径=真机截图。
5. **凭 CLAUDE.md/文档断言而未核代码**——多次 BLOCKER 来源(如"已有 NSVisualEffectView"实则全仓没有)。改前必读真实代码。
6. **QA 旁路短路真实路径**——`forceSegmentHover` 让自截图永远验不出 `@State` 失效。QA flag 不得绕过被验对象。
7. **同心半径误用致尖角**——列表中部行离 shell 远,`.containerConcentric` 算出 ~0 半径=尖角;固定 `rowCornerRadius` 才对。
8. **伪玻璃按钮**——`.glassEffect` 套在非 `Button` 上,看着能点其实不能(footer 来源胶囊)。
9. **硬编码会漂移的偏移**——派生簇用固定 offset,文案变宽即错位;必须 anchor 真实 bounds。
10. **带 caveat 提交未验视觉的 UI 改动**——把"编译过"当"完成";违反"成品=自截图迭代到满意再交终验"。

## 5. 每次 UI 改动的自检 gate(把基准变成可执行清单)

提交前**必须**全绿,任一不达标不进主干:

```
# 代码门
xcodebuild ... build                         # 真实编译,非静态分析
grep 被删符号 → 0;wc -l 超 800 文件 → 有拆分任务登记
git diff --staged --name-only | grep -v profraw/生成物   # 无脏文件入提交

# 界面门(真机截图,白天 + 夜间各一轮)
1 杀旧进程 → 启动新构建产物 → ⌘⇧V 呼出
2 screencapture -x → 裁剪放大面板
3 逐条核对 §3.2 V1–V11(真机像素为准)
4 不达标 → 改 → 重编 → 重截,直到全绿
5 真 hover/真交互验证(禁用 QA 旁路验真实路径)
6 全绿后交 Codex 无偏见复审 → 再交用户真机终验
```

**「完成」的唯一定义**:§5 全绿 + 用户真机终验通过。在此之前任何"做完了"都是违反基准。
